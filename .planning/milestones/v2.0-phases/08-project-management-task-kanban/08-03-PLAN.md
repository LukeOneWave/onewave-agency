<!-- @dnd-kit patterns from research -->
- DndContext wraps board, sensors with PointerSensor (distance: 8) + KeyboardSensor
- SortableContext per column with verticalListSortingStrategy
- useDroppable on column container for empty-column drops
- useSortable on each TaskCard
- onDragOver: move task between columns (update status in local state)
- onDragEnd: persist order + status via PATCH, optimistic with revert-on-error
- confirmedRef pattern: useRef to track last confirmed state for revert (not stale initialTasks)

       - Props: initialTasks (TaskWithAgent[]), projectId (string)
       - State: tasks (useState initialized from initialTasks)
       - Ref: confirmedRef (useRef initialized from initialTasks) — updates after each successful API call. Used for revert-on-error instead of stale initialTasks prop.

       - Collision detection: closestCorners

         a) Determine final column (status) and position (order) for the dragged task
         b) Reorder all tasks in the destination column using arrayMove from @dnd-kit/sortable
         c) Assign sequential order values (0, 1, 2, ...) to all tasks in the affected column(s)
         d) Optimistic local state update
         e) Batch PATCH calls via Promise.all to /api/tasks/[id] for each task whose status or order changed
         f) On success: update confirmedRef.current to current state
         g) On failure: revert to confirmedRef.current, show toast.error("Failed to save task order. Changes reverted.")
       - Import TASK_STATUSES, COLUMN_LABELS from @/types/project
