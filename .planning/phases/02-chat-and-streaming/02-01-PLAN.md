2. **Create src/app/api/chat/route.ts**: POST route handler that:
- Accepts JSON body: `{ sessionId, agentSlug, messages: ChatMessage[], model: string }`
- Validates API key via `settingsService.getApiKey()` — return 401 if missing with JSON `{ error: "API key not configured" }`
- Looks up agent via `agentService.getBySlug(agentSlug)` — return 404 if not found
- Creates Anthropic client with the API key
- Returns a `new Response(stream, { headers })` where stream is a `ReadableStream` that:
  - Calls `client.messages.stream({ model, max_tokens: 4096, system: agent.systemPrompt, messages })`
  - On `"text"` event: enqueues SSE `data: {"type":"text","text":"..."}\n\n`
  - On `"end"` event: reads `anthropicStream.currentMessage` for usage, enqueues `data: {"type":"done","usage":{...}}\n\n`, then closes controller
  - On `"error"` event: enqueues `data: {"type":"error","message":"..."}\n\n`, then closes controller
- Response headers: `Content-Type: text/event-stream; charset=utf-8`, `Cache-Control: no-cache, no-transform`, `Connection: keep-alive`
- Wrap the stream setup in try/catch — if Anthropic client creation fails (bad key), return 500
- After stream ends (in the "end" handler), call `chatService.addMessage` for both the user message and assistant message with token counts. Pass `sessionId` through so the route knows where to save.

3. **Create src/store/chat.ts**: Zustand store (NO persist — chat state is ephemeral per page load) with:
- State: `sessionId: string | null`, `agentSlug: string | null`, `agentName: string | null`, `messages: Array<{ id?: string; role: "user" | "assistant"; content: string }>`, `isStreaming: boolean`, `selectedModel: string` (default `"claude-sonnet-4-6"`), `error: string | null`
- Actions:
  - `initSession(sessionId: string, agentSlug: string, agentName: string, existingMessages?: Message[])` — sets session info and loads existing messages
  - `setModel(model: string)` — updates selectedModel
  - `sendMessage(content: string)` — the main action:
    1. Sets `isStreaming: true`, clears error
    2. Appends user message to messages array
    3. Appends empty assistant message (placeholder for streaming)
    4. Calls `fetch("/api/chat", { method: "POST", ... })` with sessionId, agentSlug, messages history (as ChatMessage[]), model
    5. Reads response body with `getReader()` in a while loop
    6. Maintains a buffer string. Appends each decoded chunk to buffer. Extracts complete `data: ...\n\n` segments from buffer (handles SSE chunk splitting).
    7. For `type: "text"` events: appends text to the last message's content, triggers state update
    8. For `type: "done"` events: sets `isStreaming: false`
    9. For `type: "error"` events: sets error message, sets `isStreaming: false`
    10. Wrap in try/catch: on fetch error, set error and isStreaming: false
  - `clearChat()` — resets all state
  - `stopStreaming()` — stores AbortController ref, calls abort on it, sets isStreaming: false
