       - State: `versions` (DeliverableVersion[]), `selectedOld` and `selectedNew` (version indices), `loading`, `open` (boolean to show/hide diff panel)
       - On open (lazy load): fetch `GET /api/deliverables/${deliverableId}/versions`
       - If fewer than 2 versions: show message "No previous versions to compare. Edit the deliverable to create a new version."
       - Version selector: two `<select>` dropdowns — "Old version" and "New version". Default: second-to-last as old, last as new.
       - Render `ReactDiffViewer` from `react-diff-viewer-continued`:
         ```
         <ReactDiffViewer
           oldValue={versions[selectedOld].content}
           newValue={versions[selectedNew].content}
           splitView={true}
           compareMethod={DiffMethod.WORDS}
           useDarkTheme={resolvedTheme === "dark"}
           leftTitle={`Version ${versions[selectedOld].version}`}
           rightTitle={`Version ${versions[selectedNew].version}`}
         />
         ```
       - Use `useTheme()` from `next-themes` for dark mode detection
       - Toggle button: "Compare Versions" with a `GitCompare` or `Diff` icon from lucide-react
       - Wrap the diff panel in a collapsible section (simple `open` state toggle, no animation needed)

    3. Create `src/components/chat/InlineEditor.tsx` ("use client"):
       - Props: `{ deliverableId: string; messageId: string; deliverableIndex: number; initialContent: string; onSaved: (newContent: string) => void }`
       - State: `editing` (boolean), `draft` (string), `saving` (boolean)
       - Display mode: render children (markdown content) with an "Edit" button (Pencil icon from lucide-react) that appears on hover (absolute positioned, top-right, opacity-0 group-hover:opacity-100)
       - Edit mode: `<Textarea>` with draft value, rows=12, full width. Below it: "Save" button (Check icon) and "Cancel" button (X icon).
       - On Save:
         a. Set saving=true
         b. PATCH `/api/deliverables/${messageId}` with body `{ deliverableId, content: draft }` to update Deliverable.content
         c. POST `/api/deliverables/${deliverableId}/versions` with body `{ content: draft }` to snapshot the version
         d. Call `onSaved(draft)` so parent can update local state
         e. `toast.success("Saved")` via sonner
         f. Set editing=false, saving=false
       - On Cancel: `setDraft(initialContent); setEditing(false)` — discard changes per Pitfall 4 from research
       - On error: `toast.error("Failed to save")`, keep editing mode open
