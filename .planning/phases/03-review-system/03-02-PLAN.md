1. Extend ChatState interface in src/store/chat.ts:
   - Add `deliverables: Record<string, DeliverableState>` -- keyed by `${messageId}-${deliverableIndex}`
   - Add `approveDeliverable: (messageId: string, deliverableIndex: number) => Promise<void>`
   - Add `requestRevision: (messageId: string, deliverableIndex: number, feedback: string) => Promise<void>`
   - Add `loadDeliverables: (messageId: string) => Promise<void>`

2. Implement approveDeliverable (Pattern 3 from RESEARCH.md -- optimistic UI):
   - Immediately set deliverables[`${messageId}-${deliverableIndex}`] = { status: 'approved' }
   - Then fetch PATCH /api/deliverables/${messageId} with { index, status: 'approved' }
   - On error, revert to previous state and set error

3. Implement requestRevision (Pattern 4 from RESEARCH.md -- revision auto-send):
   - Set deliverables[`${messageId}-${deliverableIndex}`] = { status: 'revised', feedback }
   - Fetch PATCH /api/deliverables/${messageId} with { index, status: 'revised', feedback }
   - Compose revision prompt: `Please revise the deliverable based on this feedback:\n\n${feedback}`
   - Call `get().sendMessage(revisionPrompt)` to auto-send as next message (REVW-04)

4. Implement loadDeliverables:
   - Fetch GET /api/deliverables/${messageId}
   - Populate deliverables state from response for persisted statuses on reload

5. Initialize deliverables: {} in default state, clear in clearChat.

6. In initSession, after setting messages, for each message with role "assistant" that has existing deliverable records, load them. Do this lazily -- only load when messages are set.

7. Implement the test stubs in src/store/__tests__/chat-review.test.ts (replace test.todo with real tests):
   - Test: approveDeliverable sets status to "approved" in state
   - Test: requestRevision sets status and calls sendMessage
   - Mock fetch globally for API calls
