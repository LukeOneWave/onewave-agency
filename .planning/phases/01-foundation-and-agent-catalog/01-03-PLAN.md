<task type="auto">
  <name>Task 2: Build settings page with API key management</name>
  <files>src/app/settings/page.tsx, src/components/settings/ApiKeyForm.tsx, src/lib/services/settings.ts, src/app/api/settings/route.ts</files>
  <action>
1. Create `src/lib/services/settings.ts`:
   - `getApiKey()`: Returns full API key string or null (server-side only)
   - `setApiKey(apiKey: string)`: Upserts key into Setting table with key="anthropic_api_key"
   - `hasApiKey()`: Returns boolean
   - `getMaskedKey()`: Returns null or masked version like "sk-ant-...xxxx" (last 4 chars)
   - `deleteApiKey()`: Removes the API key setting
   - All operations use prisma singleton
   - See 01-RESEARCH.md "Pattern 5" for reference

2. Create `src/app/api/settings/route.ts`:
   - GET: Returns `{ hasKey: boolean, maskedKey: string | null }` -- NEVER the full key
   - PUT: Accepts `{ apiKey: string }` body, validates format (starts with "sk-ant-"), saves via settingsService, returns `{ success: true, maskedKey }`
   - DELETE: Removes API key, returns `{ success: true }`
   - All endpoints use try/catch with proper error responses

3. Create `src/components/settings/ApiKeyForm.tsx`:
   - Client component ("use client")
   - On mount: fetch GET /api/settings to check if key exists, show masked key if so
   - Input field for API key (type="password") with show/hide toggle
   - "Save" button that POSTs to /api/settings
   - "Remove" button (only shown if key exists) that DELETEs
   - Success/error feedback via sonner toast
   - Validation: key must start with "sk-ant-" before allowing save
   - Loading states during API calls

4. Create `src/app/settings/page.tsx`:
   - Page title "Settings"
   - Section: "Anthropic API Key" with description explaining it's needed for agent chat
   - Renders ApiKeyForm component
   - Use shadcn Card to wrap the settings section
