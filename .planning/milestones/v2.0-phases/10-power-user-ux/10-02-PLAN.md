  <name>Task 1: Install cmdk and create CommandPalette component</name>
  <files>src/components/layout/CommandPalette.tsx</files>
  <action>
    First install cmdk:
    ```bash
    npm install cmdk
    ```

    Create src/components/layout/CommandPalette.tsx as a "use client" component:

    Props: open (boolean), onOpenChange ((open: boolean) => void)

    State: query (string), results (SearchResults | null), loading (boolean)

    Debounced search:
    - Use useRef for debounce timer (250ms)
    - On input change, clear previous timer, set new 250ms timeout that calls fetch(`/api/search?q=${encodeURIComponent(q)}`)
    - Set loading true before fetch, false after

    CRITICAL: Set `shouldFilter={false}` on root `<Command>` — filtering is done server-side, not by cmdk's built-in filter. Without this, cmdk re-filters the already-filtered API results and may hide valid matches.

    Reset on close: In onOpenChange callback, when isOpen is false, clear query and results.

    Render structure:
    ```
    Command.Dialog (open, onOpenChange, shouldFilter={false}, label="Global search")
      Command.Input (value, onValueChange with debounce, placeholder="Search agents, projects, sessions...")
      Command.List
        Command.Loading (when loading)
        Command.Empty (when query exists but no results)
        Command.Group heading="Agents" (if agents.length > 0)
          Command.Item for each agent -> navigate to /agents/{slug}
            <Users /> icon, agent name, division badge
        Command.Group heading="Projects" (if projects.length > 0)
          Command.Item for each project -> navigate to /projects/{id}
            <FolderKanban /> icon, project name
        Command.Group heading="Sessions" (if sessions.length > 0)
          Command.Item for each session -> navigate to /chat/{id}
            <MessageSquare /> icon, title ?? "Untitled session", agentName badge
    ```

    Navigate function: call router.push(url), then onOpenChange(false), then clear query/results.

    Styling with Tailwind (apply to Command.Dialog via className or wrapping div):
    - Overlay: fixed inset-0 bg-black/50 z-50
    - Dialog container: fixed top-[20%] left-1/2 -translate-x-1/2 w-full max-w-lg bg-white dark:bg-zinc-900 rounded-xl shadow-2xl border border-zinc-200 dark:border-zinc-700 overflow-hidden
    - Input: border-b border-zinc-200 dark:border-zinc-700 px-4 py-3 w-full outline-none bg-transparent
    - List: max-h-80 overflow-y-auto p-2
    - Items: px-3 py-2 rounded-lg cursor-pointer hover:bg-zinc-100 dark:hover:bg-zinc-800 flex items-center gap-2 text-sm
    - Group heading: px-3 py-1.5 text-xs font-medium text-zinc-500 uppercase tracking-wider
    - Empty/Loading: px-3 py-6 text-center text-sm text-zinc-500

    Note: cmdk Command.Dialog renders a portal by default. Style via [cmdk-dialog-overlay] and [cmdk-dialog-content] CSS attribute selectors, or use className props on the cmdk components. Check cmdk docs for which components accept className.

    Use data attributes for cmdk styling if className is not supported on all sub-components:
    ```css
    [cmdk-item][data-selected="true"] { background: ... }
    ```
    Add these as a style tag or Tailwind @layer in the component if needed.

<task type="auto">
  <name>Task 2: Integrate CommandPalette into AppShell with Cmd+K listener</name>
  <files>src/components/layout/AppShell.tsx</files>
  <action>
    Update src/components/layout/AppShell.tsx:

    1. Add imports:
    ```typescript
    import { useState, useEffect } from "react";
    import { CommandPalette } from "./CommandPalette";
    ```

    2. Add state and keyboard listener inside AppShell:
    ```typescript
    const [paletteOpen, setPaletteOpen] = useState(false);

    useEffect(() => {
      function handleKeyDown(e: KeyboardEvent) {
        if (e.key === "k" && (e.metaKey || e.ctrlKey)) {
          e.preventDefault();  // MUST preventDefault to block browser address bar
          setPaletteOpen((prev) => !prev);
        }
      }
      document.addEventListener("keydown", handleKeyDown);
      return () => document.removeEventListener("keydown", handleKeyDown);
    }, []);
    ```

    3. Render CommandPalette after the layout div (sibling, not child):
    ```typescript
    return (
      <div className="flex h-screen overflow-hidden">
        <Sidebar />
        <div className="flex flex-1 flex-col overflow-hidden">
          <Header />
          <main className="flex-1 overflow-y-auto">
            <div className="mx-auto max-w-7xl p-6">{children}</div>
          </main>
        </div>
        <CommandPalette open={paletteOpen} onOpenChange={setPaletteOpen} />
      </div>
    );
    ```

    Do NOT change any other part of AppShell. Only add the import, state, useEffect, and CommandPalette render.
  </action>
