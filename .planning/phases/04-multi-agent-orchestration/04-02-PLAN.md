  - Track `activeStreams = mission.lanes.length`
  - Define `send(data: object)` helper that JSON.stringify's and emits as `data: ...\n\n`
  - For each lane, spawn a parallel Anthropic stream:
    - `client.messages.stream({ model: mission.model, max_tokens: 4096, system: lane.agent.systemPrompt + deliverableInstruction, messages: [{ role: "user", content: mission.brief }] })`

    - On "error": `send({ type: "error", agentId, message })`, update lane status to "error", decrement activeStreams, resolve (don't reject -- other streams continue)
  - Handle request.signal abort: on abort, call `.abort()` on each Anthropic stream instance to stop consuming tokens
  - Update mission status to "done" when all streams complete
- Return Response with text/event-stream headers (same as existing chat route)

```typescript
interface OrchestrationState {
  missionId: string | null;
  brief: string;
  model: string;
  lanes: Record<string, LaneState>;
  missionStatus: "idle" | "creating" | "streaming" | "done" | "error";
  error: string | null;
  _abortController: AbortController | null;

  // Actions
  createMission: (agentIds: string[], brief: string, model: string, agents: Array<{ id: string; name: string; division: string; color: string }>) => Promise<void>;
  connectStream: (missionId: string) => Promise<void>;
  handleSSEEvent: (event: OrchSSEEvent) => void;
  stopMission: () => void;
  reset: () => void;
}
```

- `createMission`: POST to /api/orchestration, get missionId, initialize lanes from agents array (each with empty content, "pending" status), set missionStatus to "creating", then call connectStream(missionId).
- `connectStream`: fetch GET /api/orchestration/[missionId]/stream with AbortController. Read via ReadableStream reader. Parse SSE events (same buffer/split pattern as chat store). Dispatch each parsed event to handleSSEEvent. Set missionStatus to "streaming" on first event.
- `handleSSEEvent`: Switch on event.type:
  - "text": append text to lanes[agentId].content, set lane status to "streaming" if still "pending"
  - "agent_done": set lanes[agentId].status to "done"
  - "error": set lanes[agentId].status to "error", set error message
  - "mission_done": set missionStatus to "done", clear abortController
- `stopMission`: abort fetch via _abortController, set missionStatus to "done"
- `reset`: clear all state back to initial
  </action>
