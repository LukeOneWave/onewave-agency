<feature>
  <name>Dashboard Service</name>
  <files>src/lib/services/dashboard.ts, src/lib/services/__tests__/dashboard.test.ts</files>
  <behavior>
    ## getStats (DASH-01)
    - Returns { activeSessions: number, agentsUsed: number, tokensConsumed: number }
    - activeSessions = total ChatSession count
    - agentsUsed = count of distinct agentIds across ChatSession
    - tokensConsumed = sum of (inputTokens + outputTokens) across all Messages, null-coalesced to 0
    - When no data exists: returns { activeSessions: 0, agentsUsed: 0, tokensConsumed: 0 }

    ## getRecentActivity (DASH-02)
    - Accepts optional limit parameter (default 20)
    - Returns ActivityItem[] sorted by timestamp descending
    - ActivityItem = { type: "chat" | "mission" | "deliverable", description: string, timestamp: string (ISO) }
    - Chat items: "Started chat with {agent.name}"
    - Mission items: "Launched mission with {lanes.length} agents"
    - Deliverable items: "Approved deliverable from {agent.name}" or "Revised deliverable from {agent.name}"
    - Only includes deliverables with status !== "pending"
    - Merges all types, sorts chronologically, returns top N

    ## getAgentUtilization (DASH-03)
    - Returns AgentUtilization[] = { name: string, color: string, sessions: number }[]
    - Groups ChatSession by agentId, counts sessions per agent
    - Includes MissionLane sessions in count (they create ChatSessions too, so groupBy on ChatSession covers both)
    - Orders by session count descending, takes top 10
    - Resolves agent name and color from Agent table
    - When no data: returns empty array
  </behavior>
  <implementation>
    Create src/lib/services/dashboard.ts exporting dashboardService object with three methods.

    Use Prisma aggregation patterns from research:
    - `prisma.chatSession.count()` for active sessions
    - `prisma.chatSession.groupBy({ by: ["agentId"] }).then(r => r.length)` for agents used
    - `prisma.message.aggregate({ _sum: { inputTokens: true, outputTokens: true } })` for tokens
    - Null coalesce: `(sum.inputTokens ?? 0) + (sum.outputTokens ?? 0)`
    - Activity feed: three parallel queries with Promise.all, map to unified type, sort, slice
    - Utilization: groupBy on ChatSession.agentId with _count, then fetch agent names
    - Convert all Date objects to ISO strings before returning (server/client boundary safety)

    For tests, mock prisma using the same vi.mock("@/lib/prisma") pattern used in existing chat.test.ts and orchestration.test.ts. Reuse the established mock patterns in the codebase.
  </implementation>
</feature>
