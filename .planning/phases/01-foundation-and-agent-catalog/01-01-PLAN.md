  <action>
1. Create the Next.js 15 project:
   ```
   npx create-next-app@latest . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"
   ```
   If the directory is not empty, use a temp directory and move files.

2. Install ALL Phase 1 dependencies:
   ```
   npm install @prisma/client @prisma/adapter-better-sqlite3 better-sqlite3 gray-matter zod next-themes zustand sonner lucide-react react-markdown remark-gfm
   npm install -D prisma @types/better-sqlite3 vitest @vitejs/plugin-react @testing-library/react @testing-library/jest-dom jsdom
   ```

3. Initialize Prisma:
   ```
   npx prisma init --datasource-provider sqlite
   ```

4. Create `prisma/schema.prisma` with Prisma 7 patterns (CRITICAL - do NOT use Prisma 6 patterns):
   - generator: `provider = "prisma-client"` (NOT `prisma-client-js`)
   - generator: `output = "../generated/prisma"` (MANDATORY in Prisma 7)
   - Agent model: id (cuid), name, slug (unique), division, description, color, tools (optional), systemPrompt, rawMarkdown, isCustom (default false), createdAt, updatedAt
   - Setting model: id (cuid), key (unique), value, createdAt, updatedAt
   - See 01-RESEARCH.md "Agent Database Schema" for exact field definitions

5. Create `src/lib/prisma.ts` singleton with better-sqlite3 adapter:
   - Import PrismaBetterSqlite3 from `@prisma/adapter-better-sqlite3`
   - Import PrismaClient from `../../generated/prisma/client` (NOT from `@prisma/client`)
   - Store on globalThis in dev mode to prevent HMR duplication
   - See 01-RESEARCH.md "Pattern 1" for exact implementation

6. Set `.env` with `DATABASE_URL="file:./prisma/dev.db"`

7. Run `npx prisma db push` to create the database and generate the client

8. Create `vitest.config.ts` with:
   - @vitejs/plugin-react
   - test.environment: 'jsdom'
   - test.globals: true
   - resolve.alias for '@/' -> './src/'

9. Initialize shadcn/ui:
   ```
   npx shadcn@latest init
   npx shadcn@latest add button card tabs input badge separator scroll-area tooltip sheet avatar dropdown-menu
   ```

10. Update `src/app/layout.tsx` with basic HTML structure (dark mode class support via suppressHydrationWarning). Simple placeholder for now.

11. Update `src/app/page.tsx` to redirect to /agents (or show a simple "OneWave AI" landing).

   ```
   npx degit msitarzewski/agency-agents agents
   ```
   If degit fails, use: `git clone --depth 1 https://github.com/msitarzewski/agency-agents.git agents && rm -rf agents/.git`

2. Add `agents/` to `.gitignore` (these are fetched, not committed).

3. Create `prisma/seed.ts` following the pattern from 01-RESEARCH.md "Pattern 2":
   - Import gray-matter for frontmatter parsing
   - Create Zod schema: `{ name: string, description: string, color: string, tools: string.optional() }`
   - Define DIVISIONS array: design, engineering, marketing, product, project-management, spatial-computing, specialized, strategy, support, testing
   - Filter function `isAgentFile()`: skip non-.md files, skip EXECUTIVE-BRIEF.md, QUICKSTART.md, and any directories
   - For each division directory, read all valid .md files
   - Parse with gray-matter, validate with Zod
   - Generate slug from filename (strip .md)
   - Upsert into database with all fields: name, slug, division, description, color, tools, systemPrompt (content body), rawMarkdown (full file)
   - Log results: seeded count, warning count, total expected
   - IMPORTANT: Import prisma from the singleton (`../src/lib/prisma`)

4. Add seed script to package.json:
   ```json
   "prisma": { "seed": "npx tsx prisma/seed.ts" }
   ```

5. Install tsx if not already: `npm install -D tsx`

6. Run the seed: `npx prisma db seed`

7. Verify count: The seed script should log how many agents were seeded. Expected: ~61-69 valid agents.
