## Cross-Phase Integration Issues

### CRITICAL: Chat stream messageId handoff (Phase 2 → Phase 3)

**Files:** `src/app/api/chat/route.ts`, `src/store/chat.ts`

The chat SSE stream's `done` event does not include the persisted message ID. The client generates a `crypto.randomUUID()` as placeholder. When user approves/revises a deliverable, the store calls `PATCH /api/deliverables/${messageId}` with the client UUID, causing a foreign key constraint violation.

**Fix:** Include `messageId` in the SSE `done` event (like orchestration stream already does), update chat store to replace UUID with real ID.

### CRITICAL: AgentLane wrong API endpoint (Phase 4 → Phase 3)

**File:** `src/components/orchestration/AgentLane.tsx` lines 64, 82

AgentLane calls `/api/chat/messages/${messageId}/deliverables` (POST) instead of `/api/deliverables/${messageId}` (PATCH) with wrong body shape.

**Fix:** Update URL, HTTP method, and request body to match the actual deliverables API.
