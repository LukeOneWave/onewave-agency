Staggered entrance pattern (from research):
```tsx
{items.map((item, i) => (
  <div
    key={item.id}
    className="animate-in fade-in slide-in-from-bottom-2 duration-300 fill-mode-both"
    style={{ animationDelay: `${Math.min(i * 30, 240)}ms` }}
  >
    <ItemCard item={item} />
  </div>
))}
```

IMPORTANT (Pitfall 5): Do NOT apply animate-in at both page level AND list item level.
PageTransition (from Plan 01) handles page-level fade-in. List items get staggered entrance
only -- use slide-in-from-bottom-2 (NOT fade-in) to avoid double-fade conflict.
Actually, since PageTransition uses key={pathname} and fires instantly, and list items have
animationDelay, the two animations coexist because the page fade completes (300ms) while
staggered items are still cascading in. Both can use fade-in safely since they run on
different elements. BUT to keep visual clarity, list items should use a shorter slide
(slide-in-from-bottom-2 vs the page's slide-in-from-bottom-4).
